What your /etc/oratab means right now
	orcl:/home/oracle/oracle:N
	mydb2:/home/oracle/oracle:N
	prod:/home/oracle/oracle:N
	
The last column is N for all databases.
N means:

	Do NOT start this database automatically

	dbstart will skip it

	On reboot or listener restart, only manually started DB stays up

	This is exactly why you can connect to only one DB at a time.

The Fix (Very Simple & Safe)

Change N â†’ Y for BOTH databases

sudo vi /etc/oratab
		orcl:/home/oracle/oracle:Y
		mydb2:/home/oracle/oracle:Y
		prod:/home/oracle/oracle:Y

su - oracle
dbstart

If dbstart says $ORACLE_HOME not set, run:

	export ORACLE_HOME=/home/oracle/oracle
	dbstart $ORACLE_HOME

		Make it permanent (VERY IMPORTANT)

		Edit oracle user profile:

		vi ~/.bash_profile


		Add these lines:

		export ORACLE_HOME=/home/oracle/oracle
		export PATH=$ORACLE_HOME/bin:$PATH
		export ORACLE_SID=orcl


		Reload:

		source ~/.bash_profile

		Verify everything is running
		Listener status
		lsnrctl status

		Databases running
		ps -ef | grep pmon | grep -v grep
		
		
	Best Practice (Production)

	Always set ORACLE_HOME

	Always run:

	dbstart $ORACLE_HOME
	dbshut  $ORACLE_HOME
